# دليل النسخ الاحتياطي واستعادة المشروع (Backup & Restore Guide)

## 📦 النسخ الاحتياطي (Backup)

قبل إعادة تثبيت الويندوز، يجب عليك حفظ المجلدات التالية:

1. **المشروع كاملاً**: انسخ المجلد `vacant-nebula` كاملاً إلى قرص خارجي (USB) أو Google Drive.
2. **قاعدة البيانات**: تأكد من وجود ملف `backend/dev.db` (هذا الملف يحتوي على جميع البيانات).
3. **الملفات المرفوعة**: تأكد من وجود المجلد `backend/uploads` (يحتوي على صور المنتجات، المتاجر، الخ).
4. **ملفات الإعدادات البيئية (.env)**: تأكد من أن ملفات `.env` الموجودة داخل `backend` و `frontend-web` موجودة ومنسوخة.

**نصيحة:** لتوفير المساحة، يمكنك حذف مجلدات `node_modules` من كل من `backend` و `frontend-web` و `mobile-app` قبل النسخ، لأن حجمها كبير جداً ويمكن إعادة تثبيتها لاحقاً.

---

## 🛠️ استعادة المشروع (Restore)

بعد تثبيت الويندوز الجديد، اتبع الخطوات التالية لإعادة تشغيل المشروع:

### 1. تثبيت البرامج الضرورية (Tools)
- قم بتثبيت [Node.js](https://nodejs.org/) (الإصدار LTS).
- قم بتثبيت [VS Code](https://code.visualstudio.com/).
- قم بتثبيت [Git](https://git-scm.com/).

### 2. إعداد المشروع (Setup)
1. افتح مجلد المشروع في VS Code.
2. افتح "Terminal" جديدة.

### 3. تثبيت المكتبات (Install Dependencies)
يجب عليك تشغيل الأمر التالي في كل مجلد لإعادة تثبيت المكتبات:

**للخلفية (Backend):**
```bash
cd backend
npm install
```

**للموقع (Frontend Web):**
```bash
cd ../frontend-web
npm install
```

**للتطبيق (Mobile App):**
```bash
cd ../mobile-app
npm install
```

### 4. تشغيل المشروع (Running)
بعد انتهاء التثبيت، يمكنك تشغيل المشروع بسهولة باستخدام السكربت المجمع:
1. ارجع للمجلد الرئيسي: `cd ..`
2. كليك يمين على ملف `start_dev.bat` واختيار "Open" أو شغله من التيرمينال: `.\start_dev.bat`.

سيقوم هذا بتشغيل السيرفر والموقع تلقائياً.

## ⚠️ ملاحظات هامة
- إذا واجهت مشاكل في قاعدة البيانات، يمكنك تشغيل الأمر `npx prisma migrate deploy` داخل مجلد `backend` لتحديث الهيكلة.
- لتشغيل التطبيق على الهاتف، ستحتاج لتثبيت تطبيق "Expo Go" على هاتفك وتشغيل `npm start` داخل مجلد `mobile-app`.
